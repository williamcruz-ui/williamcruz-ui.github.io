O Projeto

Back-office no piloto automático.

Um dono de rede de barbearias precisa recuperar seu tempo. A solução: uma equipe de agentes de IA que assume as rotinas diárias de gestão — caixa, banco, contratos, manutenção e escalas.

“A verdade (ós números) é calculada por código determinístico. O LLM nunca inventa número — ele só explica e prioriza o que o código apurou.”

Esse é o princípio central da arquitetura. Cada agente trabalha em 2 etapas: analisar() faz a conta em Python puro (auditável, determinístico, sem alucinação); narrar() pede ao Claude para explicar os achados em linguagem natural. O LLM comunica — não decide.

A Equipe Virtual

5 agentes especializados.

Cada agente tem um papel único e segue o mesmo contrato: analisar os dados com lógica pura, depois narrar os achados com IA.

Processo

Como foi construído.

Da ideia ao produto funcional — cada decisão técnica explicada.

1

Dados mock com anomalias plantadas

O generate_data.py simula 90 dias de operação de 3 lojas com 9 barbeiros. De propósito, planta furos de caixa (~8%), divergências bancárias (~6%), MEIs vencidos, manutenções atrasadas e faltas — exatamente o que cada agente precisa encontrar.

2

Classe base com contrato analisar() + narrar()

O core/agent.py define o contrato que todo agente segue. analisar() é Python puro (lógica determinística, auditável). narrar() envia os achados pro Claude, que redige o relatório em linguagem natural. Sem API? Cai no modo offline automático.

3

5 agentes especialistas + 1 orquestrador

Cada agente tem responsabilidade única. O orchestrator.py roda todos, coleta os achados e pede ao Claude um resumo executivo cruzado — as 5 ações mais urgentes do dia, priorizadas por impacto (dinheiro e conformidade primeiro).

4

Dashboard em Streamlit com Plotly

O dashboard.py transforma os dados em painel web interativo: KPIs, gráficos de faturamento por loja, ranking de barbeiros, tabelas de divergências, calendário de manutenção e faltas/atrasos — tudo com tema escuro e navegação por seções.

5

Notificações e agendamento

O notificador.py formata o relatório para e-mail (HTML) e WhatsApp (texto com marcação). O agendador.py roda como serviço — todo dia às 7h o resumo chega na mão do dono, sem ele fazer nada.

6

Desacoplamento da fonte de dados

Só o core/tools.py conhece de onde vêm os dados. Trocar JSON por API real do sistema de gestão (Trinks, Belasis, Avec) mexe em 1 arquivo — os agentes e o orquestrador não mudam nada.

Código aberto, pronto pra estudar.

Todo o código está no GitHub com documentação detalhada. Rode localmente com python main.py e veja a equipe virtual em ação.
